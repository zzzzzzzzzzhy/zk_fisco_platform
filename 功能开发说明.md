# 竞赛平台新功能开发说明

## 已完成功能

### 1. 角色权限控制

**功能说明：**
- 用户分为两种角色：普通用户（USER）和管理员（ADMIN）
- 普通用户：可报名、提交作品、查看榜单
- 管理员：可创建竞赛、发布竞赛、冻结榜单等

**实现细节：**
- 后端：
  - User实体添加role字段
  - 创建@RequireRole注解用于接口权限控制
  - RoleInterceptor拦截器验证用户角色
  - JWT Token中包含role信息
  - 竞赛管理API（创建、更新、发布、删除）添加ADMIN权限检查

- 前端：
  - 登录时保存role到localStorage和Vuex
  - Layout.vue根据角色显示/隐藏管理后台入口
  - 榜单冻结按钮仅管理员可见

**数据库更新：**
```sql
ALTER TABLE users ADD COLUMN role VARCHAR(20) DEFAULT 'USER';
UPDATE users SET role = 'USER' WHERE role IS NULL;
```

**创建管理员账号：**
1. 注册一个普通账号
2. 执行SQL：`UPDATE users SET role = 'ADMIN' WHERE username = 'your_username';`
3. 重新登录即可获得管理员权限

### 2. 竞赛状态自动转换

**功能说明：**
- 自动根据时间转换竞赛状态：报名中(1) → 进行中(2) → 已结束(3)

**实现细节：**
- 创建CompetitionStatusTask定时任务类
- 每分钟执行一次，检查所有竞赛的时间
- 转换规则：
  - 报名中 → 进行中：当前时间 >= 提交开始时间
  - 进行中 → 已结束：当前时间 > 提交结束时间
- 主类添加@EnableScheduling启用定时任务

**测试方法：**
1. 创建一个竞赛，设置提交开始时间为当前时间+2分钟
2. 发布竞赛（状态变为"报名中"）
3. 等待2分钟后，状态自动变为"进行中"

### 3. 区块链集成（模拟实现）

**功能说明：**
- 提交作品后自动上链存证
- 榜单冻结后上链防篡改
- 测试环境使用模拟数据，生产环境可替换为真实FISCO BCOS

**实现细节：**
- MockBlockchainService：模拟区块链服务
  - storeEvidence()：模拟上链，返回ChainProof记录
  - 生成模拟的txHash、blockHeight
  - 保存到chain_proofs表

- 提交上链流程：
  1. 用户创建提交记录（chainStatus=0 未上链）
  2. SubmissionService异步调用uploadToBlockchainAsync()
  3. 构造上链数据（submissionId、userId、fileHash等）
  4. 调用mockBlockchainService.storeEvidence()
  5. 更新Submission的chainStatus=2，记录txHash、blockHeight

**数据表：**
- chain_proofs：存证记录表
  - bizType：业务类型（SUBMISSION/LEADERBOARD等）
  - bizId：业务ID
  - dataHash：数据哈希
  - txHash：交易哈希（模拟生成）
  - blockHeight：区块高度（递增）

**测试方法：**
1. 提交一个作品
2. 等待2-3秒（模拟预检耗时）
3. 刷新提交列表，chainStatus应变为"已上链"
4. 查看chain_proofs表，应有对应记录

### 4. 榜单功能（模拟数据）

**功能说明：**
- 查看竞赛榜单（模拟数据）
- 管理员冻结榜单并上链
- Merkle Tree防篡改机制
- 验证榜单完整性

**实现细节：**

#### 4.1 Merkle Tree工具类
- MerkleTreeUtil.calculateMerkleRoot()：计算Merkle根
- 自底向上构建二叉树，两两配对计算SHA256
- 用于验证数据完整性

#### 4.2 榜单服务
- getLeaderboard()：获取榜单（模拟10条数据）
- freezeLeaderboard()：冻结榜单
  1. 生成榜单快照
  2. 计算Merkle Root
  3. 上链存证
  4. 保存到leaderboards表
- verifyLeaderboard()：验证榜单完整性

#### 4.3 榜单API
- GET /leaderboards/{competitionId}：获取榜单
- POST /leaderboards/{competitionId}/freeze：冻结榜单（ADMIN）
- GET /leaderboards/{competitionId}/frozen：获取冻结榜单
- GET /leaderboards/{competitionId}/verify：验证榜单

#### 4.4 前端榜单页
- 竞赛详情页的"榜单"标签
- 显示排名、用户名、得分
- 前三名显示奖牌图标
- 管理员可见"冻结榜单"按钮
- 显示链上存证信息（txHash、blockHeight、merkleRoot）

**测试方法：**
1. 进入任意竞赛详情页
2. 点击"榜单"标签，查看模拟数据
3. 管理员登录后，点击"冻结榜单"
4. 确认后，榜单冻结成功，显示链上信息

### 5. 竞赛图片上传

**功能说明：**
- 管理员在创建/编辑竞赛时可以上传两张图片
- 参考Taikai网站设计，实现双图片展示效果
- **Position 1 (封面图片 - coverImage)**：
  - 用途：竞赛列表页的卡片背景大图
  - 默认图片：11.jpeg
- **Position 2 (详情图片 - detailImage)**：
  - 用途1：竞赛列表页的圆形Logo图（叠加在封面底部）
  - 用途2：竞赛详情页的头部横幅图
  - 默认图片：22.jpg
- 不上传时使用默认图片

**实现细节：**

#### 5.1 后端实体
- Competition实体添加coverImage和detailImage字段
- 字段类型为TEXT，支持base64编码或图片URL

#### 5.2 前端创建表单
- CompetitionForm.vue添加两个图片上传组件
- 支持图片预览功能
- beforeImageUpload()：验证文件类型和大小（最大5MB）
- 读取图片并转换为base64编码
- resetCoverImage()：重置为默认图片
- loadDefaultImages()：加载默认图片或已有图片

#### 5.3 前端列表页（双图片设计）
- Competitions.vue采用类似Taikai的双图片设计
- **Position 1 (封面图)**：作为卡片背景大图
  - getCoverStyle()方法动态设置背景图片
  - 使用coverImage（默认11.jpeg）
  - 优先级：自定义图片 > 默认图片 > 渐变背景
- **Position 2 (Logo图)**：圆形图片叠加在封面底部
  - getLogoImage()方法获取Logo图片
  - 使用detailImage（默认22.jpg）
  - 尺寸：80px直径，白色边框，阴影效果
  - 位置：margin-top: -40px实现一半叠加效果

#### 5.4 前端详情页
- CompetitionDetail.vue显示详情图片（Position 2）
- 页面顶部添加横幅图片区域（400px高度）
- getDetailImage()方法返回图片URL

**默认图片：**
- 封面图片：frontend/src/images/11.jpeg (推荐尺寸：1200x400px)
- 详情图片：frontend/src/images/22.jpg (推荐尺寸：1200x600px)

**测试方法：**
1. 进入管理后台 → 创建竞赛
2. 上传两张图片：
   - 封面图片（推荐1200x400px）：用于列表页背景
   - 详情图片（推荐正方形或1200x600px）：用于列表页Logo和详情页横幅
3. 保存后，在竞赛列表页查看效果：
   - 卡片背景显示封面图片（Position 1）
   - 卡片底部叠加圆形Logo图片（Position 2）
4. 进入竞赛详情页，查看头部横幅图片（Position 2）

**数据库更新：**
```sql

ALTER TABLE competitions ADD COLUMN detail_image TEXT;
```

## 使用流程

### 管理员操作流程

1. **注册账号**
   - 注册一个普通账号
   - 数据库执行：`UPDATE users SET role = 'ADMIN' WHERE username = 'your_username';`
   - 重新登录

2. **创建竞赛**
   - 进入"管理后台" → "竞赛管理"
   - 点击"创建竞赛"
   - 填写竞赛信息、时间、奖金配置
   - 保存为草稿（status=0）

3. **发布竞赛**
   - 在竞赛列表中点击"发布"
   - 状态变为"报名中"（status=1）
   - 系统将自动根据时间转换状态

4. **冻结榜单**
   - 竞赛结束后，进入竞赛详情页
   - 点击"榜单"标签
   - 点击"冻结榜单"
   - 榜单冻结并上链

### 普通用户操作流程

1. **注册并登录**
   - 默认角色为USER

2. **报名竞赛**
   - 浏览竞赛列表
   - 点击竞赛查看详情
   - 点击"立即报名"

3. **提交作品**
   - 进入"我的提交"
   - 点击"提交作品"
   - 选择已报名的竞赛
   - 上传文件（自动计算哈希并上链）

4. **查看榜单**
   - 进入竞赛详情页
   - 点击"榜单"标签
   - 查看排名和链上存证信息

## 注意事项

1. **角色权限**
   - 普通用户无法访问竞赛管理接口
   - 尝试访问会返回"权限不足"错误

2. **定时任务**
   - 每分钟自动执行一次
   - 生产环境建议改为每5分钟或每小时

3. **区块链**
   - 当前使用模拟实现
   - 生产环境需替换为真实FISCO BCOS调用
   - 修改BlockchainService，注释掉MockBlockchainService的调用

4. **模拟数据**
   - 榜单数据是模拟的，每次访问生成相同的10条记录
   - 生产环境需连接Evaluation表获取真实评测结果

5. **数据库**
   - 执行database_update.sql前请备份数据库
   - role字段默认值为'USER'
